웹 표준 기반의 웹 문서의 구조는 담당하는 HTML, 표현(디자인)을 담당하는 CSS, 동작을 담당하는 것은 JavaScript로 구성됩니다.

이렇게 3가지 요소가 유기적으로 결합하여 작동하게 되면 웹 문서가 가벼워지며, 유지보수 및 수정시에도 간편하고 빨리 처리가 가능합니다.

Table Layout VS. CSS Layout

CSS Layout의 장점

  • 웹 표준 방식의 CSS Layout은 콘텐트가 논리적으로 선형화 됩니다.
  • CSS Layout은 Table Layout에 비하여 파일의 용량을 50% 이상 절감해 줍니다.
  • 사람과 컴퓨터 모두 이해하기 쉬운 구조가 됩니다.
  • 유지보수가 용이해지고 유지보수 비용을 절감해줍니다.

HTML의 구조

  • HTML문서의 확장자는 .html 혹은 .htm이다.
  • 문서의 맨 앞에는 DTD(Document Type Definition)를 선언한다.
  • 최상위 태그는 <html>을 사용한다.
  • <head>태그는 문서를 설명하는 태그로 제목이나 키워드와 같은 정보를 담는다.
  • <body>태그는 문서의 내용이 위치한다.

DTD(Document Type Definition)

브라우저는 DTD의 선언을 확인하고 브라우저 모드를 결정.

DTD를 올바르게 선언하지 않을 경우 CSS가 의도하지 않은 결과로 랜더링 될 수 있으므로 올바른 문서 형식을 선언해주는 것은 다양한 웹 브라우저에 따른 랜더링 차이를 최소화할 수 있는 가장 기본적이면서 중요한 것.

block-level VS. inline element

  • block-level element
    • 한줄에 하나의 엘리먼트만 표시되는 종류의 엘리먼트
    • 다른 인라인 엘리먼트 뿐 아니라 블록 레벨 엘리먼트도 컨텐츠로 포함할 수 있다.
    • DIV, H1~H6, P, FORM, UL, LI, ADDRESS, BLOCKQUOTE, FIELDSET, TABLE, HR 등
  • inline element
    • 한줄에 여러 개의 엘리먼트가 표시되는 종류의 엘리먼트
    • 일라인 엘리먼트만 포함 할 수 있고, 블록레벨 엘리먼트의 자식이어야 한다.
    • span, a, img, em, strong 등

이미지

<img src="이미지가 위치하는 URL" alt="대체텍스트" width="폭" height="높이" longdesc="링크" />
<img src="http://farm7.staticflickr.com/6234/7abbf0_s.jpg" width="50" height="50" alt="이미지" />
  • src : source의 약자로 이미지가 위치하는 URL을 기술한다.
  • alt : alternative의 약자로 한국어로는 대체 텍스트라고 부른다. 이미지가 로딩되기 전이나, 이미지를 로딩할 수 없는 경우 이미지의 위치에 텍스트가 표시된다. 시각장애인을 위한 장치와 검색엔진에서도 사용된다.
  • width, height : 이미지의 크기
  • longdesc : 이미지와 관련된 링크

CSS(Cascading Style Sheets)

HTML이 문서의 구조와 컨텐츠를 담당한다면

CSS는 옷과 액서사리와 같은 꾸미는 역할을 담당합니다.

p{
    font-size: 110%;
    font-family: garamond, sans-serif;
}
h2{
    color: red;
    background: white;
}
.highlight{
    color: red;
    background: yellow;
    font-weight: bold;
}

#test_id {
    color: blue;
    background: white;
}

CSS 기본 구조

Selector {Property: Value; Property: Value;}

h1 {color:red; font-weight:bold;}

HTML에 CSS 추가하기

  • inline 방식
    • tag에 style을 직접 기술하는 방식
    • selector가 필요 없다
    • 코드가 많아지고, 구조와 표현이 분리라는 CSS의 취지에 벗어난다.
    • 유지보수가 어렵다.
  • style tag 방식
    • style tag에 기술하는 방식
    • 다른 페이지에 영향을 미치지 않고 손쉽게 적용할 수 있다.
    • 구조와 표현이 분리된다.
    • 공통 페이지에 적용이 어렵고, cache를 사용할 수 없다.
  • 외부 파일 방식
    • CSS를 별도 파일로 분리해서 링크하는 방식
    • 공통 페이지에 적용할 수 있고, cache를 사용하므로 경제적이다.
    • 유지보수가 용이하다.

inline 방식

  • tag에 style을 직접 기술하는 방식
  • selector가 필요 없다
  • 코드가 많아지고, 구조와 표현이 분리라는 CSS의 취지에 벗어난다.

style tag 방식

  • style tag에 기술하는 방식
  • 다른 페이지에 영향을 미치지 않고 손쉽게 적용할 수 있다.
  • 구조와 표현이 분리된다.
  • 공통 페이지에 적용이 어렵고, cache를 사용할 수 없다.

<html>
	<head>
		<title>HTML&CSS Basic</title>
		<style>
		h1 {
			color:red;
		}
		</style>

	<body>
		<h1>사내 강의</h1>
		<p>HTML, CSS를 배위보아요!</p>
	</body>
</html>

외부 파일 방식

  • CSS를 별도 파일로 분리해서 링크하는 방식
  • 공통 페이지에 적용할 수 있고, cache를 사용하므로 경제적이다.
  • 유지보수가 용이하다.

	<head>
		<title>HTML&CSS Basic</title>

	</head>

		<h1>사내 강의</h1>
		<p>HTML, CSS를 배위보아요!</p>
	</body>

Selector

CSS가 적용될 태그를 지정

Selector {Property: Value; Property: Value;}

h1 {color:red; font-weight:bold;}

ID Selector

#idName {Property: Value;}

  • tag의 ID값을 기준으로 식별
  • 주로 한번만 사용되는 레이아웃이나 JS와의 연동을 위해 사용
<style>
	#wrap {
		float:left;
		width:960px;
	}
</style>
<div id="wrap">
	Contents
</div>

Class Selector

.className {Property: Value;}

  • tag에 공통적으로 style을 적용하기 위해 사용
  • 공백으로 구분된 여러 개의 class가 표시될 수 있음.
<style>
	.memo {
		color:red;
	}
	.box {
		border: 1px solid red
	}
</style>
<div id="memo box">
	red
</div>

Type Selector

tagName {Property: Value;}

  • 특정 태그명을 가진 엘리먼트 전체를 제어하기 위해 사용
<style>
	h1 {
		color:red;
	}
</style>
<h1>
	사내교육
</h1>

Descent Selector(하위 선택자)

Selector Selector {Property: Value;}

  • 특정 엘리먼트의 하위에 나오는 엘리먼트를 선택하는데 사용
  • 부모가 먼저 선언하고 자식을 나중에 선언
  • 공백을 통해 부모와 자식을 구분

<style>
	.logo a {
		color:red;
	}
</style>
<div class="logo">
	<a href="http://www.jobkorea.co.kr/">잡코리아</a>
</div>

Selector 중복 적용

Selector, Selector {Property: Value;}

  • 하나의 CSS선언을 여러 엘리먼트에 적용하고자 할때 사용
  • 선택자와 선택자를 콤마(,)로 구분

<style>
	.logo, memo {
		color:red;
	}
</style>
<div class="logo">
	<a href="http://www.jobkorea.co.kr/">잡코리아</a>
</div>
<p class="momo">
	memo
</p>
